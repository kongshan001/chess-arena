## 阶段一: MVP 打磨 (第1-4周)

### 🎯 目标
- 验证核心玩法体验
- 确认可玩性 + 创新性
- 产出可玩 Demo

### 👥 团队配置
| 岗位 | 人数 |
|------|------|
| 策划 | 2人 |
| 美术 | 3人 |
| 交互 | 1人 |
| 前端 | 3人 |
| 后端 | 3人 |
| QA | 2人 |
| PM | 1人 |

### 📦 MVP 版本内容

#### 核心玩法
- ✅ 8人经典对战
- ✅ 20+ 基础棋子
- ✅ 7种基础羁绊 (人类/精灵/兽族/亡灵/战士/法师/刺客)
- ✅ 技能系统 (普攻+主动+终极)
- ✅ 经济系统 (金币/利息/连胜)
- ✅ 段位系统 (青铜-王者)

#### 美术 (MVP)
- 基础棋子立绘 (5个)
- 棋盘 UI
- 1个战斗场景
- 基础特效

#### 技术
- Pixi.js 渲染
- WebSocket 房间
- 帧同步战斗

### ✅ 验收标准
1. 核心战斗循环流畅
2. 棋子羁绊逻辑正确
3. 帧率 ≥ 30fps (主流机型)
4. 试玩留存率 ≥ 60%

## 阶段二: 美术资源迭代 (第5-10周)

### 🎯 目标
- 视觉品质升级到头部水平
- 完成全部棋子美术资源
- 场景多样化

### 📦 工作内容

#### 美术 (第5-10周)
| 周次 | 内容 |
|------|------|
| 5-6 | 棋子立绘补全 (20个) + 特效优化 |
| 7-8 | 场景迭代 (3个) + 粒子特效 |
| 9-10 | UI 重构 + 动效完善 + 音效接入 |

#### 前端 (配合)
- 资源加载优化
- 动画效果增强
- 性能优化 (目标 60fps)
