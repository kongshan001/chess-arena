# 战斗服务架构设计

## 服务架构

### 1. 整体架构
```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   Gateway   │───▶│  Room Svc   │───▶│ Battle Svc  │
│   (WS/HTTP) │    │  (房间管理)  │    │  (战斗逻辑)  │
└─────────────┘    └─────────────┘    └─────────────┘
                         │                   │
                   ┌─────▼─────┐       ┌─────▼─────┐
                   │  Redis    │       │   Game    │
                   │  (状态存储)│       │   State   │
                   └───────────┘       └───────────┘
```

### 2. 核心服务

#### Room Service（房间服务）
- `createRoom`: 创建游戏房间
- `joinRoom`: 加入房间
- `leaveRoom`: 离开房间
- `ready`: 玩家准备
- `startGame`: 开始游戏

#### Battle Service（战斗服务）
- `initBattle`: 初始化战斗
- `frameSync`: 帧同步
- `action`: 玩家操作
- `endBattle`: 战斗结束

### 3. 帧同步方案
- 固定帧率：30fps
- 服务器 authoritative
- 客户端预测+服务器校验
- 断线重连支持

### 4. 战斗逻辑
- 服务端计算所有伤害
- 技能释放顺序
- 羁绊效果计算
- 胜负判定

### 5. API设计
- WebSocket长连接
- JSON协议
- 心跳保活
- 断线重连
