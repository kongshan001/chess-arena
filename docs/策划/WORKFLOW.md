# 策划岗 - 工作流程与规范

## 📋 上下游关系

```
上游: PM(需求输入) → 策划(设计) → 下游: 前端/后端(实现) → QA(验收)
```

## 📝 每日工作流程

| 时间 | 工作内容 | 产出 |
|------|----------|------|
| 09:00-09:30 | 晨会 + 今日计划 | 工作计划表 |
| 09:30-12:00 | 核心设计工作 | 设计文档 |
| 12:00-13:30 | 午休 | - |
| 13:30-15:00 | 设计完善 + 评审 | 评审记录 |
| 15:00-17:30 | 与下游对接 | 需求答疑 |
| 17:30-18:00 | 日总结 + 记录 | 工作日志 |

## 📁 文档清单

### 必须记录
- 工作记录 (每日)
- 进度 (每周)
- 规划 (每阶段)
- 调研报告 (关键决策前)
- 学习成长 (每月)

### 模板位置
```
team/策划/工作记录/    # 每日工作记录
team/策划/进度/       # 进度跟踪
team/策划/规划/       # 短期/长期规划
team/策划/调研报告/   # 市场/竞品调研
team/策划/学习成长/   # 技能提升/复盘
```

## 🔄 与上下游对接规范

### 上游 (PM) 对接
- 每日确认需求优先级
- 阻塞及时升级
- 变更需书面确认

### 下游 (前端/后端) 对接
- 设计文档必须包含: 流程图 + 接口说明 + 数据结构
- 每日站会同步进度
- 48小时内响应技术问题
- 变更需提前48小时通知

## 📈 关键指标

| 指标 | 要求 |
|------|------|
| 文档完整性 | 100% |
| 需求澄清响应 | ≤ 2小时 |
| 变更提前期 | ≥ 48小时 |
| 评审参与率 | 100% |
